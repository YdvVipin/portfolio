How It Works

A 4-step pipeline from raw test data to executable automation scripts.

1

Read CSV Steps

Parse manual test step data from CSV files. Normalize columns, extract test scenarios and expected results.

2

LLM Processing

Feed structured test data into Groq's Llama 3.1 model via LangChain prompt templates with output parsing.

3

Generate BDD Features

AI produces Gherkin feature files with Given/When/Then steps mapped from the original manual test cases.

4

Generate Selenium Scripts

Produce executable Python Selenium scripts with proper locators, waits, and assertions ready to run.
