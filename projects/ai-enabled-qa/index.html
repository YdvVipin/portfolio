The Challenge

Why This Exists

Manual QA test creation is slow, fragile, and doesn't scale. AI-only approaches hallucinate 20-30% of the time. I built something better.

The Problem

  • Manual test writing takes hours per feature
  • Selectors break on every UI change
  • Pure LLM-generated tests fail 20-30% of the time
  • No feedback loop — same mistakes repeated
  • Test maintenance becomes a full-time job

My Solution

  • Record once, generate production-ready tests in seconds
  • 9-tier selector strategy with confidence scoring
  • Deterministic first — AI only for enhancement (not core logic)
  • Self-learning from failures via vector DB pattern store
  • Self-healing locators with automatic fallbacks
System Design

3-Layer Smart Pipeline

Deterministic layers handle the core logic. AI layers enhance — not generate — the output. This inverted approach gives reliability + intelligence.

L1

Selector Resolver

core/recording/selector_resolver.py

Receives raw DOM element bundles (8-10 selectors each) from the browser recorder and scores them through a 9-tier deterministic priority chain. Filters out dynamic IDs (React Aria, UUIDs). Always produces the most stable selector available.

Deterministic Confidence Scoring Dynamic ID Filtering
L2

Playwright Code Builder

core/generators/playwright_code_builder.py

Template-based TypeScript code generation using Playwright-native locators (getByTestId, getByRole, getByLabel). Handles 10+ event types: clicks, inputs, selects, drag-drop, file uploads, dialog interactions, keyboard events, form submissions.

Deterministic TypeScript Output Template Engine
2.5

AI Polish & Fallback

core/agents/playwright_generator_agent.py

LangGraph-powered agent that polishes the deterministic output with better variable names, page object patterns, and retry logic. Serves as fallback when templates can't handle edge cases. Only ~10-15% of tests need this layer.

AI-Enhanced LangGraph Agent Fallback Path
L3

Multi-LLM Enrichment

core/generators/ai_enrichment.py & mcp_servers/a2a_orchestration_wrapper.py

3-LLM debate pattern: Claude analyzes code quality, GPT-4o suggests coverage improvements, Grok generates edge-case assertions. Weighted voting produces consensus-enhanced test code with BDD/Gherkin scenarios.

Multi-LLM Claude + GPT + Grok BDD Generation

Capabilities

Key Features

End-to-end QA automation — from browser recording to test execution, with Jira integration and self-learning.

Browser Recording

Point-and-click recording with live event capture. Supports clicks, inputs, drag-drop, file uploads, dialogs, keyboard events. No code needed to create tests.

Deterministic Code Gen

Template-based TypeScript generation that always produces valid Playwright code. Uses native locators like getByRole and getByTestId for maximum stability.

Multi-LLM Orchestration

3-LLM consensus debate: Claude, GPT-4o, and Grok each analyze generated tests. Weighted voting produces enhanced, battle-tested output.

Self-Healing Tests

4-agent LangGraph pipeline automatically fixes failing tests. Analyzes errors, applies corrections, validates, and learns from each fix for future improvements.

Self-Learning System

Stores successful patterns in Weaviate vector DB. Error capture, auto-correction, and prompt evolution reduce failure rates over time through continuous learning.

Interactive Dashboard

React 18 + TypeScript dashboard with real-time analytics, agent metrics, code comparison views, and live recording controls — all via WebSocket.

Jira & Zephyr Integration

Bidirectional Jira sync: generate test cases from tickets, push results back. Zephyr Scale integration with full folder hierarchy and test execution linking.

Dockerized Microservices

6-service Docker Compose stack: FastAPI, PostgreSQL, Redis, Weaviate, n8n workflows, and React frontend — all with health checks and persistent volumes.

18 MCP Agents

Specialized agents for BDD generation, test prioritization, coverage analysis, code intelligence, and learning — communicating via Model Context Protocol.

AI Agents

Self-Healing Pipeline

When a test fails, a 4-agent LangGraph pipeline automatically diagnoses, fixes, validates, and learns — without human intervention.

AGENT 01
Code Analyzer

Parses failing test with Tree-Sitter AST. Identifies error type: broken selector, assertion mismatch, timeout, or logic error.

AGENT 02
Code Fixer

Applies targeted fix: selector update, assertion correction, retry logic injection, or wait strategy adjustment.

AGENT 03
Quality Validator

Runs the fixed test against the target app. Verifies pass/fail status. Checks for regressions in other test steps.

AGENT 04
Pattern Learner

Extracts the fix pattern and stores it in Weaviate vector DB. Future tests benefit from accumulated knowledge.

Generated Test Output — TypeScript + Playwright
import { test, expect } from '@playwright/test';

test('User can complete checkout flow', async ({ page }) => {
  await page.goto('https://shop.example.com');

  // L1: data-testid selector (confidence: 100)
  await page.getByTestId('product-card').first().click();

  // L1: role + aria-label (confidence: 95)
  await page.getByRole('button', { name: 'Add to Cart' }).click();

  // L2: Template-generated navigation
  await page.getByRole('link', { name: 'Checkout' }).click();

  // L3: AI-enriched assertions (via Multi-LLM consensus)
  await expect(page.getByTestId('cart-count')).toHaveText('1');
  await expect(page.getByRole('heading')).toContainText('Checkout');
  await expect(page).toHaveURL(/checkout/);
});

Key Technical Decisions

Architectural trade-offs and design philosophy behind this project.

Deterministic First, AI Second

Template-based code generation handles 85-90% of cases reliably. AI only enhances — never generates core logic. This inverted approach gives enterprise-grade reliability while still benefiting from LLM intelligence.

Multi-LLM Over Single-LLM

A single LLM has blind spots. By debating between Claude, GPT-4o, and Grok with weighted voting, the system produces more robust output than any individual model alone.

Selector Bundling Strategy

Storing 8-10 selectors per element instead of one means when a primary selector breaks, the system can instantly fallback to alternatives without re-recording. This is what makes self-healing actually work.

Graceful Degradation

Every subsystem degrades independently. If Redis is down, tests still generate. If LLMs are unreachable, deterministic templates still work. No single point of failure takes down the whole platform.
