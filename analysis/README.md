# JIRA Data Analysis Project

## Overview
Comprehensive analysis of JIRA ticket data to extract QA insights and improve testing processes.

## Setup
```bash
pip install -r requirements.txt
python jira_analysis.py
```

## Generated Charts
- Priority Distribution (Pie Chart)
- Issues Over Time (Line Chart)
- Resolution Time by Priority (Box Plot)
- Issue Types Distribution (Bar Chart)
- Component Analysis (Stacked Bar Chart)

## Key Findings
- 40% of issues are bugs
- Average resolution time: 12.5 days
- Critical issues resolved 60% faster than low priority
- Frontend component has highest bug count
